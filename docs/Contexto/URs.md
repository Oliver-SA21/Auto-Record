## Roles de Usuario (URs)
A continuación se detallan los roles definidos para el sistema **Auto Record** y las funciones específicas que desempeña cada uno dentro de la plataforma:
| Rol | Descripción |
| :--- | :--- |
| **Administrador** | Encargado de gestionar las rutas, dar de alta unidades y supervisar el desempeño general. |
|  | Gestión de Catálogos sobre la Alta, baja y modificación de unidades (autobuses) y rutas. |
|  | Control de Personal del Registro y asignación de credenciales para choferes y checadores. |
|  | Monitoreo Estratégico de la Visualización de todas las unidades en tiempo real sobre el mapa. |
|  | Análisis de Datos en la Generación de reportes, cumplimiento de horarios y estadísticas de incidencias. |
| **Checador** | Registro de Paso para validar la hora exacta en la que una unidad llega a un punto de control. |
|  | Control de Intervalos para notificar al chofer si va adelantado o retrasado respecto al tiempo ideal de la ruta. |
|  | Reporte de Frecuencia para registrar la saturación de pasajeros en las unidades en puntos clave. |
|  | Gestión de Turnos para confirmar el relevo de choferes en las terminales si es necesario. |
| **Chofer** | Usuario que opera la unidad, registra el inicio/fin de jornada y las incidencias en ruta. |
|  | Inicio/Cierre de Turno para la apertura de sesión vinculada a una unidad y ruta específica. |
|  | Navegación Asistida para la visualización del mapa con la ruta trazada y paradas autorizadas. |
|  | Reporte de Incidencias para el envío de alertas inmediatas por tráfico, fallas mecánicas o emergencias. |
|  | Estado de la Unidad para actualizar si la unidad está "En Servicio", "En Mantenimiento" o "Fuera de Servicio". |
| **Usuario Pasajero** | Persona que consulta las rutas disponibles, horarios y ubicación aproximada de las unidades. |
